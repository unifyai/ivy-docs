Containers#

Collection of containers for handling ESM input and output data

class ivy_memory.geometric.containers.ESMCamMeasurement(img_mean, cam_rel_mat=None, img_var=None, validity_mask=None, pose_mean=None, pose_cov=None, dev_str=None)[source]#

Bases: Container

__init__(img_mean, cam_rel_mat=None, img_var=None, validity_mask=None, pose_mean=None, pose_cov=None, dev_str=None)[source]#

Create esm image measurement container

Parameters:

  • img_mean (Array) – Camera-relative co-ordinates and image features [batch_size, timesteps, height, width, 3 + feat]

  • cam_rel_mat (Optional[Array]) – The pose of the camera relative to the current agent pose. (default: None) Default is identity matrix [batch_size, timesteps, 3, 4]

  • img_var (Optional[Array]) – Image depth and feature variance values, assumed all zero if None. (default: None) [batch_size, timesteps, height, width, 1 + feat]

  • validity_mask (Optional[Array]) – Validity mask, for which pixels should be considered. (default: None) Assumed all valid if None [batch_size, timesteps, height, width, 1]

  • pose_mean (Optional[Array]) – The pose of the camera relative to the current agent pose, (default: None) in rotation vector pose form. Inferred from cam_rel_mat if None. [batch_size, timesteps, 6]

  • pose_cov (Optional[Array]) – The convariance of the camera relative pose, in rotation vector form. (default: None) Assumed all zero if None. [batch_size, timesteps, 6, 6]

  • dev_str (Optional[str]) – Device string to use, default is to use img_mean. (default: None)
